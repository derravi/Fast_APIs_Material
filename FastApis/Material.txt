#HTTP Status Code
-> Whenever client is sent the request to the server then the server is give the response to you request with the Status code.
->This Status code is show the request is succesfull or note like that.

Status Code     Explanation

->2xx       :-  Success        -> The request is succesfuly recived and preoced.
->3xx       :-  Redirection    -> Further action need to be taken.
->4xx       :-  Client Error   -> Somethinf is wron with the request from the client.
->5xx       :-  Server Error   -> Something went wrong on the server.

-> Let see some Different Different Error of the server.

Code------Movement------------------------------Description-------------------------------------------

200         OK              ->Standard success                 ->A GET OF POST succeeded
201       Created           ->Resource created                 ->After a POST that creates something
204      No Content         ->Success, but no data returned    ->After a DELETE request
400     Bad Request         ->Malformed or invalid request     ->Missing field, wrong data type
401     Unauthorized        ->No/invalid authentication        ->Login required
403      Forbidden          ->Authenticated, but no permission ->Logged in but not allowed
404      Not Found          ->Resource doesn't exist            ->Patient ID not in DB
500   Internal Server Error ->Generic failure                   ->Something broke on the server
502     Bad Gateway         ->Gateway (like Nginx)              ->failed to reach backend  
503  Service Unavailable    ->Server is down or overloaded

#Query Parameter
->This Query Parameter is use for the Filter the data directly from the URL.
-> Query parameters in FastAPI are the values added to the URL after a question mark (?) to send extra information to the server. They are mainly used for filtering, searching, or sorting data.

->Query parameters are not part of the main URL path but are optional details that modify the API response. Each parameter is written as a key-value pair, and multiple parameters are separated by an ampersand (&).

-> Let see one Example of this.
Ex. /patient?city=Delhi&sort_by=age
